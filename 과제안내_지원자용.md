# 젠테라 개발자 채용 실무 프로젝트 안내

**문서번호**: XT-HR-2607-01
**제목**: 개발자 채용 2차 전형(실무 프로젝트) 안내
**수신**: 지원자 귀하
**발신**: 주식회사 젠테라 채용 담당

---

## 1. 안내

1차 면접에 응해 주셔서 감사합니다. 2차 전형으로 아래와 같이 실무 프로젝트를 안내드립니다.
본 전형은 당사의 실제 업무(제조 데이터 처리·검증·시각화)를 축소한 프로젝트를 직접 수행하는
과정으로, 완성도보다 데이터에 대한 판단과 검증 과정을 중점적으로 살펴봅니다.

- **제출 기한**: 본 안내 수신 후 24시간 이내
- **AI 도구 사용**: 허용하며 권장합니다 (Claude, ChatGPT, Copilot 등 제한 없음). 단, 제5항의
  "AI 활용 기록"을 함께 제출해 주시기 바랍니다.
- **기술 스택**: 자유. 프론트엔드는 React 계열을 권장하며, 그 외 언어·도구는 편하신 것을
  사용하시면 됩니다.

## 2. 프로젝트 개요

당사는 제조 데이터(입고·출고·재고 실사)를 다루는 B2B 시스템을 개발·운영하고 있습니다.
첨부된 CSV 4개 파일은 한 자재 창고의 2026년 5~6월 데이터입니다 (합성 데이터).

| 파일 | 내용 |
|---|---|
| items.csv | 품목 마스터 (품목코드, 품명, 단위, 박스입수, 단가) |
| receipts.csv | 입고 기록 (영수증ID, 일자, 품목코드, 수량, 단가) |
| issues.csv | 출고 기록 (출고ID, 일자, 품목코드, 수량) |
| stock_snapshot_0531.csv | 5월 31일 실사 재고 (품목코드, 실사 수량) |

유의: 실무 데이터와 마찬가지로 본 데이터에는 품질 문제가 포함되어 있습니다. 무엇을
발견하였고 어떻게 처리하였는지가 평가의 핵심입니다.

## 3. 수행 과제

1. **데이터 정합 처리** — CSV를 읽어 정제하고, 발견한 데이터 문제와 각각의 처리 규칙
   (제외/보정/유지)을 README에 표로 정리해 주십시오. 정해진 정답이 있는 것이 아니며,
   판단과 그 근거를 확인합니다.
2. **재고 계산** — 품목별 일별 재고를 계산해 주십시오. 기준: 6월 30일 시점 품목별 현재고
   = 5월 31일 실사 수량 + 6월 입고 − 6월 출고. 5월 데이터는 검증용으로 활용해 주십시오
   (5월 거래 누적과 5월 31일 실사의 일치 여부를 대조하고, 차이가 있는 품목의 목록을
   산출물에 포함).
3. **대시보드 화면** — 다음을 표시하는 웹 화면을 구현해 주십시오.
   - 품목별 현재고 목록 (6월 30일 기준, 단위와 박스입수 반영)
   - 품목 선택 시 일별 재고 추이 그래프
   - 데이터 이상 목록 화면 (발견한 품질 문제 건들을 사유와 함께 표시)
4. **검증 자료** — 산출 수치의 근거를 제시해 주십시오. 최소 요건:
   - 합계 대사표 (총 입고 수량, 총 출고 수량, 제외 건수와 사유별 집계)
   - 핵심 계산에 대한 자동 테스트 1건 이상
5. **수요 예측** — 7월 1주차(7/1~7/7) 품목별 예상 출고 수량을 제시해 주십시오. 방법은
   자유이나 다음 두 가지를 필수로 포함합니다.
   - 방법 선택의 근거 (검토한 대안 포함)
   - 자체 검증: 동일한 방법으로 "5월 데이터만으로 6월을 예측"한 뒤 실제 6월과의 오차를
     수치로 제시
6. **운영 설계 서술** (코드 불필요, README에 서술) — 본 시스템이 실제 운영된다고 가정할 때:
   ① 매일 새 거래 CSV가 도착하는 경우 전체 재계산 없이 반영하기 위한 변경 방안
   ② 데이터가 1일 100만 행 규모로 증가할 경우 현 설계에서 먼저 한계에 도달하는 지점과
   대응 방안. 각 10줄 내외.
7. **AI 활용 기록** — 사용한 도구, 지시한 작업, AI 산출물에서 확인·수정한 사항의 요약
   (형식 자유, 10줄 내외). AI 사용 자체는 평가에 불리하지 않으며, 검증 없이 사용한
   경우에만 불리하게 반영됩니다.

## 4. 제출물

     바랍니다.

   운영 설계 서술, AI 활용 기록
4. **자동 테스트 실행 방법과 결과 요약** (README에 포함)

## 5. 평가 기준

아래 기준으로 평가하며, 기준을 사전에 공개하는 것은 당사의 방침입니다.

1. 데이터 품질 문제의 발견 수준과 처리 판단의 합리성
2. 자기 산출물에 대한 검증 여부 (대사표, 테스트, 불일치 목록, 예측 자체 검증)
3. 화면이 데이터를 정확하고 이해하기 쉽게 전달하는지
4. 판단의 근거 제시 (방법 선택 이유, 대안 비교, 운영 설계의 현실성)
5. AI 산출물에 대한 이해와 통제

완성도가 부족한 경우에도 수행 범위와 미완 사항을 사실대로 기재한 제출물을 미완성의
은폐보다 높게 평가합니다.
